# 苏适酒店全自助体验设计

> **核心判断**：苏适酒店是**全自助无人值守酒店**。真正的前台是**微信小程序**。人不在现场，Agent 的服务对象重新分层：**C 端直面客人**，B 端服务运营。

---

## 1. 项目定位重新表述

| 维度 | 最初理解 | 真实定位 |
|------|---------|---------|
| 前台形态 | 员工+AI 辅助 | **客人直接跟小程序对话，员工基本不在场** |
| Agent 服务对象 | 员工 | **客人（C 端） + 运营（B 端）双层** |
| 主战场 | 后台控制台 | **微信小程序** |
| 人力结构 | 标准酒店班组 | 清洁 + 工程 + 远程客服（省 60% 人力） |
| 核心竞争力 | 成本优势 / 一致体验 / 24h 随时自助 |

### 场景细化

#### 2.1 自助办理入住

**客人旅程**：到店 → 大堂自助机 或 小程序入口 → 扫身份证 → 人脸比对 → 选房号 → 签电子入住单 → 收到电子房卡

**异常分支**：
- OCR 识别失败 → 人工客服远程接入（企微视频）
- 人脸不匹配 → 强制人工核验
- 公安联网核查未通过 → 拒绝办理 + 上报
- 支付押金失败 → 引导重试 / 切换支付方式

**涉及系统**：
- 微信小程序 / 大堂自助机（Android 平板定制）
- **阿里云 OCR**（身份证识别）
- **腾讯云人脸识别**（活体检测 + 1:1 比对）
- **公安治安旅业系统**（省级，必接）
- **PMS（捷信达）**：订单查询 + 办入住 API
- **门锁系统**：下发电子房卡（密码/蓝牙钥匙）
- **短信网关**：推送电子房卡信息
- **微信支付**：押金预授权

**合规要点**：
- 身份证图像、人脸特征值加密存储（SM4）
- 公安联网 30 分钟内上报
- 押金金额阈值告警（防薅羊毛/洗钱）
- 夜间（23:00-次日 6:00）的自助办理触发额外风控

---

#### 2.2 小程序开房门

**客人旅程**：到房门口 → 小程序"开门"按钮 → 门锁打开

**两种技术路径**：

| 方案 | 原理 | 优缺点 |
|------|------|-------|
| **蓝牙直连** | 手机蓝牙对门锁广播 | ✅不依赖网络，锁电量低 ❌要求手机蓝牙开启，范围内 |
| **云端下发** | 小程序→服务端→门锁 Wi-Fi/4G | ✅远程开门（比如授权家人） ❌网络差时失败 |

**推荐：双模并行**，蓝牙优先，失败降级云端。

**权限控制**：
- 只有"当前在住客人"能开对应房号
- 入住期间每次开门记日志
- 小程序内显示"最近开门记录"（透明 + 防家庭纠纷）

---

#### 2.3 房内客控

**客人旅程**：小程序 → 当前房间 → 滑块调温度/亮度，开关场景

**控制项**：空调温度/风速、灯光（顶灯/床头/卫生间/氛围灯）、窗帘、音乐、勿扰/请清洁模式

**智能叠加**（客房管家 Agent 加持）：
- 入住时自动设到客人偏好温度（画像已记录）
- 语音指令"我冷了"→ Agent 解析 → 空调 +2°C
- 离房 30 分钟自动切节能模式

**协议落地**：
- 客控主机（菲迪/蓝豆）暴露 Modbus TCP 或 MQTT
- 建议用 **MQTT Broker 统一北向**，小程序只认 MQTT 事件

---

#### 2.4 扫码打咖啡

**客人旅程**：扫咖啡机二维码 → 小程序弹出房号确认 → 选品 → 出杯

**权益识别**：
- 标准房型：每房每天免费 2 杯
- 豪华房型：全天免费
- 会员等级额外加量
- 超出后按 ¥ 挂账

**供应链联动**：
- 咖啡豆/牛奶/杯子实时库存上报
- 低于阈值 → 供应链 Agent 触发补货
- 机器故障 → 工程运维 Agent 派工单

---

#### 2.5 扫码寄存行李

**客人旅程**：扫行李柜二维码 → 小程序确认房号 → 柜门打开 → 存包 → 小程序显示取件码

**机型推荐**：
- **德生智能寄存柜**（商用主流）
- 或自研：电磁锁柜 + 二维码 + 摄像头

**异常处理**：
- 取件超时（离店 > 24h 未取）→ 清洁员转存储藏室 + 短信通知
- 柜体故障 → 扫旁边备用柜 + 工程工单

---

#### 2.6 洗衣机预约

**客人旅程**：小程序"洗衣房" → 看 3 台设备状态 → 预约下一轮 → 洗完推送

**核心字段**：
- `剩余 N 分钟`
- `当前使用者（脱敏为房号末位）`
- `我的预约位置（队列第 X）`

**付费/免费规则**：
- 住 3 晚以上免费 1 次
- 超出 ¥10/次，挂房账

---

#### 2.7 订房 / 续住

**续住场景最重要**：客人住到一半想延一晚

**体验**：小程序 → 点"续住一晚" → 显示可续价格（可能折扣） → 一键确认

**收益 Agent 加持**：
- 看剩余房量决定给不给续住折扣
- 旺季可能反而要加价（系统自动判断）

---

#### 2.8 取早餐

**三种模式**：

| 模式 | 场景 | 硬件 |
|------|------|------|
| A. 餐厅自助扫码 | 到餐厅扫码签到 | 餐厅入口扫码器 |
| B. 房间送餐（高端房） | 小程序点→服务员送 | 员工工单 |
| C. 保温柜自提 | 小程序订→到时间取柜 | 智能保温取餐柜 |

推荐 A + C 组合，B 是增值服务。

---

#### 2.9 客需送物（6 小件）

**6 小件定义**：牙刷、梳子、拖鞋、剃须刀、浴帽、沐浴盖（各地略有差异）

**两种交付方式**：
- **客需柜自取**（主推）：大堂 / 每层客需柜，扫码取 → 完全自助
- **员工送到房间**（兜底）：小程序下单 → 工单给值班员工 → 送达

**统计维度**：供应链 Agent 看每日消耗，数字质检看客人满意度

---

#### 2.10 发票

**客人旅程**：离店后 → 小程序"我的订单" → 申请发票 → 填税号 → 电子发票邮箱接收

**技术**：标准，用诺诺或百望 API，基本即用

---

#### 2.11 停车

**车主流程**：
1. 预约车位（可选）：小程序录车牌
2. 入场：道闸摄像头识别车牌 → 自动放行
3. 出场：识别车牌 → 自动结算 → 挂房账 或 微信支付

**集成深度**：
- 简单版：接捷停车云平台
- 深度版：对接本地车场系统，做"房费含免费 12h 停车"之类的权益

---

#### 2.12 住中 FAQ + 求助

**AI 首层**：
- 小程序右下角气泡 → 在线客服 Agent 问答（Qwen + 酒店知识库）
- 解决 80% 常见问题（Wi-Fi 密码、周边吃饭、几点退房等）

**人工兜底**：
- "转人工"按钮 → 企微客户联系群 → 远程客服接入
- **紧急按钮**（医疗/安全）→ 直通店长 + 报警联动

## 5. 建议新增 Agent

### 🧳 客需调度 Agent（新）

**职责**：统一管 12 种自助服务请求的排队 / 分配 / 完成闭环

**为什么独立**：原来客需散在客房管家 / 质检 / 工程多处，全自助下客需量暴增，需要独立调度。

**典型任务**：
- 客人申请 6 小件 → 判断客需柜还是员工送 → 派单 → 追踪完成
- 早餐 10 份送房 → 按楼层分批 → 分给值班员
- 咖啡机缺豆 → 供应链补 + 标记不可用 → 客人请求时推迟

### 🎟️ 会员权益 Agent（新）

**职责**：每次客人请求时实时判断权益

**典型查询**：
- 客人 A 扫咖啡机 → 权益 Agent 查：今天还能免费几杯？
- 客人 B 申请洗衣 → 判断：含几次免费？
- 客人 C 停车 → 判断：免费时长？

**数据源**：PMS 订单 + 会员系统 + 房型权益表 + 营销活动

## 7. 小程序 BFF API 设计（关键粒度）

小程序对 BFF 的调用主要分 4 类：

### 7.1 会话与身份
```
POST /bff/auth/wechat-login     # 微信授权登录
GET  /bff/auth/me                # 当前用户信息（含当前在住订单）
```

### 7.2 订单 / 房间
```
GET  /bff/hotels                 # 门店列表（含当前活动/可订房型）
POST /bff/orders                 # 创建订单
POST /bff/orders/:id/check-in    # 自助办入住
POST /bff/orders/:id/check-out   # 自助退房
POST /bff/orders/:id/extend      # 续住
GET  /bff/orders/:id/invoice     # 申请发票
```

### 7.3 住中服务（核心！）
```
POST /bff/room/:roomNo/unlock           # 开门
POST /bff/room/:roomNo/control          # 客控（温度/灯光/场景）
GET  /bff/room/:roomNo/status           # 房内状态（温度/设备/客控）

POST /bff/coffee/:machineId/brew        # 打咖啡
POST /bff/laundry/book                  # 洗衣预约
GET  /bff/laundry/status                # 洗衣机实时状态
POST /bff/luggage/open                  # 行李柜开箱
POST /bff/breakfast/order               # 早餐下单
GET  /bff/breakfast/pickup-code         # 取餐码
POST /bff/amenity/request               # 6 小件申请
POST /bff/parking/bind                  # 绑车牌
```

### 7.4 智能客服
```
POST /bff/chat/ask               # 问 AI（Qwen 流式返回）
POST /bff/chat/transfer-human    # 转人工
POST /bff/chat/sos               # 紧急求助
```

## 8. 关键数据流（3 条核心链路）

### 8.1 自助入住链路

```
小程序 → BFF
  → [阿里云 OCR] 身份证 OCR
  → [腾讯云人脸] 活体 + 1:1 比对
  → [公安治安] 旅业实名核查
  → [PMS 捷信达] 查订单 + 办入住 → 分配房号
  → [门锁系统] 下发电子房卡 / 生成临时密码
  → [短信/小程序消息] 推送房卡信息给客人
  → [事件总线] 推送 "guest.checked_in" 事件
    → 客房管家 Agent：按画像预设房间温度
    → 宾客画像 Agent：标记本次入住行为
    → 数字营销 Agent：触发欢迎券/活动推送
```

### 8.2 控客控链路

```
小程序 "开空调到 22°C"
  → BFF
  → [权限校验] 当前客人在 208 房？
  → [客房管家 Agent] 意图理解 + 参数校验
  → [IoT 网关] MQTT publish → hotel/room/208/ac/set
  → [客控主机] 执行 → 状态回读
  → BFF 收到确认 → 返回小程序
```

### 8.3 续住链路

```
小程序 "我想续一晚"
  → BFF
  → [PMS 捷信达] 查当前订单 + 查明日房态
  → [收益 Agent] 算续住价格（可能折扣）
  → 返回小程序：¥389 续一晚
  → 客人确认 → [微信支付]
  → [PMS 捷信达] 延长订单 + 更新房态
  → [门锁系统] 延长房卡有效期
  → [事件总线] 推送续住事件给各 Agent
```

## 9. 实施优先级与节奏

分两条线并行推进：

### 线 A · B 端运营监控（当前在推）
已完成：12 agent 控制台 + 指挥大屏 + IOT 设备中枢占位
下一步：PMS 捷信达对接 + 依赖与数据 tab 补全其余 8 个 agent

### 线 B · C 端客人自助（新线，需先决策硬件）

**Phase 1 — MVP 三件套（1-2 月）**：
- 自助入住（必须）
- 小程序开门（必须）
- 客控控制（必须）

这三样没搞定，酒店根本无法自助运营。

**Phase 2 — 体验增强（3-4 月）**：
- 订房/续住
- 客需送物
- 智能客服问答
- 发票

**Phase 3 — 差异化服务（5-6 月）**：
- 咖啡机
- 洗衣机预约
- 早餐
- 行李柜
- 停车

## 10. 合规风险提示（全自助模式特有）

| 风险 | 规避 |
|------|------|
| 客人"刷脸即入住"可能被冒用身份 | 必须活体检测 + 公安联网核查双闸门 |
| 自助办理夜间风险高（醉酒/未成年同行） | 夜间模式增加强制视频客服接入 |
| 押金争议（设备未归还/损坏） | 入住时小程序拍照取证 |
| 小程序被羊毛党刷免费咖啡 | 每次请求走会员权益 Agent |
| 陌生设备登录 | 新设备开门二次人脸 |
| 客人隐私（开门记录） | 只给本人看，员工查询需审批记日志 |
